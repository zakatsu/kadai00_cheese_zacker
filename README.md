# ①課題名
チーズバーガーアカデミー level.2

## ②課題内容（どんな作品か）
チーズアカデミーを仕様書に忠実に作成しつつ、こだわりのUIを反映できるCSSを追記した入門編level2です。

デプロイしている場合はURLを記入（任意）

## ④アプリのログイン用IDまたはPassword（ある場合）
- なし

## ⑤工夫した点・こだわった点
- 「UIこうなったら良いな」を調べながら反映してみました。
  - サイト全体の背景色をチーズっぽく
  - スクロール時の上部メニュー枠の固定（`position: sticky;`）
  - ナビゲーションメニュー選択時のハイライト表示
  - 写真とかぶっているタイトルに背景色を追加してみやすく（`z-index`）
  - ニュースのカード化・選択時のハイライト（`object-fit`、`box-shadow`）

- レスポンシブ対応
  - ナビゲーションの文字がはみ出るサイズ（1080px以下）でハンバーガーメニューの設置（⭐一番こだわり）
    - 「ハンバーガーメニュー＞チーズバーガー?!」と思い、ハンバーガーメニューをチーズバーガーの画像にしました笑
    - ※チーズバーガーのイラストをClaude × Figmaで作成！
    - 画面サイズが小さくなり、ロゴとハンバーガーメニューの画像が画面幅内で競合したら、両者の比率が均等に小さくなるように設定
  - SP表示で崩れないように、SP時の表示を`@media`で制御
    - 画面幅より大きい要素があると崩れることを学びました
    - COURSE欄・NEWS欄・CONTACT欄はスマホ用特別表示で、見やすくなったと思います
  - SP時に改行位置が崩れて読みづらくならないように、SP時のみ適用される改行を追加！（プチこだわり）

## ⑥難しかった点・次回トライしたいこと（又は機能）
- Contact欄のテーブルを作った時の、`<tr>`や`<th>`、`<td>`の関係性、それらの`margin` / `padding`の考え方が直感的に分からず苦労しました。
- レスポンシブ対応をしようとするとCSSが冗長化するので、そうならないような記述の工夫が難しいと思いました。

## ⑦フリー項目（感想、シェアしたいこと等なんでも）
- 一切HTML/CSSがわからない状態から、入れてみたいUIを調べながら少しずつ表現できるようにできて嬉しかったです！
- AIとの付き合いは以下の役割分担でやってみました。
  - 初学者なので、構造を理解するために一度指示書通りに自力で作成
  - その後の自分のこだわりはAIに聞きながら進行
    - AIに直接コピペしてもらうと反映箇所がわかりづらいし理解できないので、「反映はしないで提案だけして」という風に指示出し
    - 新しい構文は調べたり、AIに直接聞いたりして、なるべく理解してから反映するように意識
